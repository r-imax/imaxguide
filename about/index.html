Project Mission

The IMAX Theatre Database is an open-source project dedicated to creating the most comprehensive and accurate database of IMAX theatres worldwide. Our goal is to provide detailed technical information about IMAX installations, including screen specifications, projector types, and capabilities.

Why this project exists: A comprehensive list was previously available on Wikipedia but was removed due to policy disagreements. This site exists to provide a consolidated, freely accessible database without ads, with enhanced sorting and filtering capabilities.

Technical Details

The IMAX Theatre Database is built with modern web technologies and designed for performance, accessibility, and maintainability.

Technology Stack:

  • Frontend: HTML5, CSS3, vanilla JavaScript (ES6+) - no framework dependencies
  • Libraries: PapaParse for CSV processing, modern browser APIs
  • Data Storage: Regional CSV files organized by continent for better performance
  • Deployment: GitHub Pages with automated GitHub Actions workflows
  • Validation: Comprehensive Python validation pipeline ensuring data quality
  • Architecture: Modular JavaScript classes with shared functionality across pages

Key Features:

  • Enhanced Filtering System: Screen size (Large >20m, Standard 15-20m, Small <15m), format (1.43:1 True IMAX, 1.90:1 IMAX Digital, Dome), and film capability filters
  • Regional Pages: Dedicated pages for Americas, Europe, Asia, Africa, and Oceania with optimized performance
  • Hierarchical Filtering: Cascading region ‚Üí country ‚Üí province/state ‚Üí city filter system
  • Real-time Search: Instant search across all theatre data fields
  • Data Export: CSV export functionality with timestamps
  • Responsive Design: Mobile-first design that works perfectly on all devices
  • Unit Conversion: Metric/Imperial toggle for international accessibility
  • Performance Optimized: Data caching and lazy loading for fast browsing

Data Format:

Theatre data is stored in CSV format with standardized columns. This approach makes it easy for contributors to add data using spreadsheet applications and ensures the database remains maintainable by the community.
